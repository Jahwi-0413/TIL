## JIRA

### 1. issue 관리
- Epic
  - 큰 틀
  - story, task들을 묶는 단위
  - 주 단위로 해결해야 되는 작업 정의
- story
  - 사용자 관점 기능
  - 누가 어떤 일을 해서 어떤 목적을 달성할지
- task
  - user story 외의 기술적, 관리적 업무
  - 설계, 서버 설치 등
- sub-task
  - story, task를 더 작은 단위로 나눔

### 2. Story Point
- 사용자 스토리를 실제로 동작하는 기능으로 구현하는게 얼마나 어려운지를 나타내는 값
- 팀 마다 다르겠지만 1SP는 4H로 설정하는게 일반적
- 1SP에 해당하는 작업을 기준점으로 잡고 작업에 SP 할당
  - ex) 로그인 기능을 1SP로 기준점으로 잡고 채팅 기능을 5SP로 할당한다
- velocity
  - 스프린트당 얼마나 많은 스토리 포인트를 획득(사용자 스토리 구현)하는지를 나타냄
  - 스프린트를 반복하며 SP 할당량을 바꿔가며 팀의 벨로시티 확인
  - 벨로시티가 대략적으로 추정이 되면 일정이 변경될 때 작업 할당에 유용하게 쓰일 수 있음
